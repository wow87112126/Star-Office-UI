# Star Office UI 公网部署资料清单（阿里云域名）

## 目标
把 Star Office UI 安全地发布到公网，同时尽量降低被扫、被打、被误配置的风险。

## 你需要提供给我的数据

### A. 域名与 DNS 信息
1. **你的主域名**
   - 例如：`example.com`
2. **准备给这个项目使用的子域名**
   - 推荐单独用一个子域名，不要和主站混用
   - 例如：`pet.example.com` / `office.example.com`
3. **域名 DNS 现在托管在哪里**
   - 阿里云云解析 DNS
   - Cloudflare
   - 其他 DNS 服务商

### B. 服务器信息
4. **准备承载页面的服务器公网 IP**
5. **服务器操作系统**
   - Ubuntu / Debian / CentOS / Windows 等
6. **服务器是否已经有反向代理**
   - Nginx / Caddy / Apache / 没有
7. **服务器上是否已经部署了别的网站**
   - 这决定是否复用现有 Web 服务

### C. HTTPS 证书方案
8. **你希望怎么拿 HTTPS 证书**
   - 推荐：Let's Encrypt 自动签发
   - 若你已有证书，也可以提供证书来源

### D. 网络暴露与安全策略
9. **安全组当前开放了哪些端口**
   - 至少告诉我 80 / 443 / 22 的状态
10. **SSH 登录方式**
   - 用户名
   - 是否密钥登录
   - 是否禁用密码登录
11. **是否允许我建议你把应用只监听 127.0.0.1**
   - 推荐：应用只本机监听，由 Nginx/Caddy 反代到公网

### E. 部署偏好
12. **你更希望哪种部署方式**
   - 方式 1：Nginx + Python 进程
   - 方式 2：Caddy + Python 进程
   - 方式 3：Docker / Docker Compose
   - 方式 4：先临时 Cloudflare Tunnel，后面再正式上域名
13. **这个页面是否只给你自己看**
   - 如果是，建议加一层访问控制
   - 例如 Basic Auth / Cloudflare Access / IP 白名单

## 我推荐的安全落地方案

### 推荐架构
**域名 → 80/443 → Nginx/Caddy → 127.0.0.1:19000 → Star Office UI**

### 为什么这样最稳
1. **不要把 19000 直接暴露到公网**
2. 应用只监听本机或仅供反向代理访问
3. 统一由反代处理：
   - HTTPS
   - 域名
   - 安全响应头
   - 访问日志
   - 限流（可选）
4. 如果只给你自己看，可以再加：
   - Basic Auth
   - Cloudflare Access
   - WAF/CDN

---

## 安全底线（非常重要）

1. **不要直接把 Flask 开发服务裸露到公网**
2. **不要开放 19000 到 0.0.0.0/0**
3. **必须启用 HTTPS**
4. **ASSET_DRAWER_PASS 必须改强密码**
5. **FLASK_SECRET_KEY 必须改强随机值**
6. **SSH 尽量使用密钥登录，关闭弱密码**
7. **安全组只开放必要端口：22/80/443**
8. **如果只是你自己用，建议额外加一层访问认证**
